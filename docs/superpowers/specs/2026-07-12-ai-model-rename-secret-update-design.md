# AI 模型重命名与密钥替换设计

## 目标

允许管理员在编辑 AI 模型时修改数据库模型名称，并通过输入新 API Key 替换现有密钥。

## 编辑语义

- 编辑表单中的模型名称允许输入，不再禁用。
- 前端更新请求同时提交 `original_model` 和新的 `model`。
- `original_model` 用于定位数据库原记录，`model` 是更新后的真实上游模型名。
- 名称未变化时执行普通更新。
- 名称变化时更新同一条数据库记录，不采用“删除后新建”。
- 新名称已被其他环境或数据库模型使用时拒绝更新。
- 历史报告中的旧模型名称不修改。

## API Key 语义

- 后端和管理员接口继续不返回完整 API Key。
- 编辑时 API Key 输入框保持空白。
- 输入新值表示加密并替换数据库中的旧密钥。
- 留空表示保留数据库中的旧密钥。
- 输入框辅助文案明确说明替换行为。

## 配置来源限制

- 数据库来源模型支持在线重命名。
- 环境变量和旧版配置来源的模型不能在线重命名，因为重启后原环境变量模型会再次出现。
- 环境变量模型仍可在线修改 Endpoint、说明、启停、默认状态或写入新的数据库密钥覆盖；模型名字段保持只读。

## 后端修改

- `AIModelMutationRequest` 增加可选 `original_model`。
- `AIModelRegistry.save_override()` 校验原模型来源、名称冲突和密钥继承。
- `AIModelRepository` 增加按原模型名更新同一条记录的方法。
- 重命名时若未输入新密钥，沿用原数据库密文。
- 更新完成后刷新运行时模型注册表。

## 前端修改

- `AIModelInput` 增加 `original_model`。
- 编辑数据库模型时启用模型名输入；编辑环境变量模型时保持禁用。
- 保存编辑时附加原模型名。
- API Key 标签和提示说明“输入新 SK 将替换，留空保留当前密钥”。

## 错误处理

- 原模型不存在：返回 400 和明确错误。
- 新模型名为空：返回 400。
- 新模型名已存在：返回 400。
- 环境变量模型尝试重命名：返回 400。
- 新 API Key 加密失败：保持原记录不变并返回安全错误。

## 测试与验证

- 后端测试数据库模型重命名后旧名称消失、新名称可解析。
- 后端测试重命名留空 API Key 时密钥不变。
- 后端测试新 API Key 被加密替换。
- 后端测试名称冲突和环境变量模型重命名被拒绝。
- 前端测试数据库模型名称可编辑、环境变量模型名称只读、更新请求包含 `original_model`。
- 浏览器实际把数据库模型改名后再改回，确认列表、编辑弹窗和连接测试正常。
- 运行后端测试、前端测试、前端构建和 `git diff --check`。

## 不在本次范围

- 不回显或解密展示完整 API Key。
- 不修改历史报告。
- 不改变公开模型选择接口格式。
