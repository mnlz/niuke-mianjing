# AI 模型永久 ID 与多渠道设计

## 目标

允许同一个真实模型名通过多个中转站接入，并使用数据库自增数字 ID 永久定位每条模型配置。

## 数据模型

- 复用 `ai_model_configs.id` 作为永久 `BIGINT AUTO_INCREMENT` 主键。
- 删除 `model` 上的唯一索引，允许多条记录使用相同真实模型名。
- 新增必填 `channel_name`，用于区分同名线路，例如“主线路”和“备用线路”。
- 不对 `model + endpoint` 增加唯一约束；不同密钥或用途可能需要共享相同 Endpoint。
- `model` 始终保存传给上游接口的真实模型名。
- 现有记录迁移时将渠道名称设为“默认线路”，数字 ID 保持不变。

## API 与服务

- 模型列表返回 `id`、`model`、`channel_name`、描述和默认状态。
- 后台编辑、删除、设为默认和连接测试全部使用数字 `id` 定位。
- AI 报告生成请求使用 `model_id`；注册表按 ID 解析 Endpoint、密钥和真实模型名。
- 前端仅展示 `模型名 · 渠道名称`，不向普通用户暴露 Endpoint。
- 密钥继续只允许提交新值进行替换，接口永不返回明文。
- 不存在、停用或无效的模型 ID 返回明确的请求错误。

## 报告引用

- `ai_analysis_reports` 新增可空 `model_id`，新报告同时保存 `model_id` 和真实模型名。
- 现有历史报告保留原模型名，`model_id` 为空，不做猜测式回填。
- 删除模型配置不删除历史报告；历史报告仍可展示生成时保存的真实模型名。

## 配置兼容

- 数据库模型使用永久正整数 ID，是多渠道配置的正式来源。
- 旧 `.env` 单模型配置继续作为无数据库记录时的兜底，不支持多渠道管理。
- 当前数据库中已有的模型记录直接沿用现有 ID，不重新编号。

## 前端

- 后台表格增加数字 ID 和渠道名称列，行键改为 ID。
- 新增和编辑表单都可修改真实模型名、渠道名称、Endpoint 和密钥。
- AI 分析向导保存 `selectedModelId`，同名模型按渠道名称区分。

## 测试

- 仓库迁移可删除模型名唯一索引并保留已有 ID。
- 注册表可同时加载两个同名、不同 ID 的模型，并按 ID 精确解析。
- 更新、删除和测试连接只影响指定 ID。
- 前端类型、请求参数和选择状态均使用数字 ID。
- 保留一次真实模型连接测试，验证解析出的模型配置能够完成请求。

## 非目标

- 不实现负载均衡、自动故障转移、权重路由或渠道计费。
- 不向用户展示完整 Endpoint 或密钥。
- 不自动合并或去重同名模型配置。
