# AI 分析静态页面设计

## 目标

基于当前 OfferLens AI 分析前端，生成 4 个可独立打开、可互相跳转的静态 HTML，用于完整展示和评审 AI 分析产品流程。

## 交付文件

文件放在 `niuke-mianjing-frontend/public/`：

- `ai-analysis-static-home.html`：AI 分析首页。
- `ai-analysis-static-create.html`：报告配置引导页。
- `ai-analysis-static-sample-report.html`：公开报告示例。
- `ai-analysis-static-reports.html`：我的报告中心。

每个文件内嵌自身所需的 CSS 和 JavaScript，不依赖构建工具、React 或后端 API。四页只复用现有 `/offerlens.svg` 和 `/company-logos/` 静态图片；图片不可用时仍显示文字。

## 统一视觉与导航

- 延续当前 OfferLens 白底、蓝色主色、细网格背景和报告卡片风格。
- 顶部导航包含 AI 分析、我的报告、新建分析和静态中文用户名。
- 页面间链接直接使用 4 个静态 HTML 文件名。
- 桌面和移动端均可正常浏览，无外部字体、图表库或 JavaScript 依赖。

## 页面内容

### AI 分析首页

- 保留主视觉、数据概览、六种分析模式、使用步骤和行动按钮。
- 展示内联 SVG 六维雷达图和三条内测反馈。
- “免费生成”“开始配置”和报告类型卡跳转配置页。
- “查看报告示例”跳转静态示例报告。

### 报告配置引导页

- 提供 6 个可点击步骤：分析类型、目标公司、目标岗位、面经范围、个人简历、确认生成。
- 公司仅使用卡片选择，不显示重复下拉框。
- 面经支持勾选，最多展示 8 条已选状态。
- 简历支持粘贴文本；本地识别邮箱、手机号和字数，展示可编辑解析结果，修改后撤销确认。
- 未确认简历时，最终生成按钮提示用户先确认；确认后跳转报告中心。
- 所有内容使用内置假数据，不上传文件或发送请求。

### 公开报告示例

- 展示完整假报告，而不是报告中心。
- 包含综合评分、岗位信号、能力雷达、匹配项、能力缺口、简历建议、面试问题和行动计划。
- “配置我的报告”跳转配置页。

### 我的报告中心

- 使用 3 份内置假报告展示统计、类型筛选、关键词搜索和报告列表。
- 查看操作打开静态报告详情弹层。
- 删除只修改当前页面内存中的假数据，刷新后恢复。
- 下载操作生成本地 Markdown 文本文件。
- “新建分析”跳转配置页。

## 非目标

- 不连接登录、报告、岗位、面经或简历解析后端。
- 不实现真实 AI 生成、PDF 解析、账号持久化或跨页面表单持久化。
- 不增加 npm 依赖或共享静态资源文件。

## 验收标准

- 4 个 HTML 均可通过浏览器独立打开。
- 四页主要导航和行动按钮跳转正确。
- 配置页 6 步、简历确认校验可操作。
- 报告中心搜索、筛选、查看、删除和下载可操作。
- 页面无控制台错误，移动端布局不横向溢出。
