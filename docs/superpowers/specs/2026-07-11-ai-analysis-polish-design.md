# AI 分析页面体验优化设计

## 目标

根据浏览器批注优化账号展示、AI 分析首页和报告配置向导，不新增第三方依赖或额外 AI 调用。

## 默认中文昵称

- `app_users` 新增可空的 `display_name VARCHAR(64)`。
- 注册时生成“意象词 + 动物 + 四位数字”的昵称，例如 `星河海豚·4821`。
- 词库固定在后端，使用 `secrets.choice` 和 `secrets.randbelow`。
- 登录或读取账号时，如果历史账号没有昵称，则生成并保存。
- 注册、登录、`me` 接口返回 `display_name`。
- 前端会话保存昵称，导航优先显示昵称，不再直接显示邮箱。

## AI 分析首页

- 删除“数据有出处 / 结论可执行 / 组合够灵活”三张说明卡。
- 替换为左右布局：左侧展示六维求职能力雷达，右侧展示 3 条用户评价和报告使用数据。
- 雷达图使用内联 SVG，评价为固定示例文案，不引入图表库和评价后台。
- 保留现有实时数据统计条和六种分析方式。

## 报告名称

- “四合一作战地图”全局替换为“全景求职研判”。
- 同步修改首页卡片、向导选项、预览、示例报告、报告中心标签及后端完整报告提示词中的相关展示文案。
- 内部 `report_type=full` 保持不变，避免接口和历史报告迁移。

## 配置向导精简

- 目标公司步骤删除顶部单选/多选 Select，只保留公司卡片网格。
- 单公司场景点击卡片切换公司。
- 多公司场景点击卡片进行多选，继续保留至少选择两家的校验。
- 删除页面标题右侧“我的报告”按钮；顶部导航中的“我的报告”保留。

## 简历解析结果

- PDF 解析接口继续返回完整文本，不增加 AI 解析调用。
- 上传成功后显示解析结果卡：文件名、文本字数、检测到的邮箱和手机号。
- 完整解析文本放在明确标记的可编辑文本框中。
- 文本发生修改后，确认状态重置。
- 用户点击“确认使用此简历”后才能在完整分析、简历诊断或反向匹配场景进入最终生成。
- 直接粘贴文本同样可以确认；非简历场景仍允许跳过。
- 第六步显示“已确认 / 待确认”，生成前再次校验。

## 测试

- 后端测试昵称格式、历史账号昵称补齐和 API 返回字段。
- 前端测试简历联系人提取、确认状态校验和新报告名称。
- 运行后端 pytest、compileall、前端测试和生产构建。
- 浏览器验证导航昵称、首页雷达与评价、向导精简、简历修改后重新确认。

## 不做

- 用户自定义昵称编辑。
- AI 简历结构化解析。
- 真实用户评价采集和统计后台。
