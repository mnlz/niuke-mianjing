# AI 分析页面浏览器批注修正设计

## 目标

修正报告配置页、示例报告页和报告中心的 7 处浏览器批注问题，保持 AI 分析首页右侧的现有报告预览不变。

## 配置向导

- 报告类型卡片改为稳定的“编号与英文代号 / 中文名称 / 描述”纵向结构，避免窄列中标题重叠。
- 顶部导航调整为三段式布局：左侧品牌、中间页面导航、右侧账号与退出；移动端保留可操作入口。
- 删除侧栏和底部的键盘快捷键提示，同时删除对应的全局键盘监听，避免隐藏行为。
- 向导从 6 步扩展为 7 步：分析类型、目标公司、目标岗位、面经范围、个人简历、分析模型、确认生成。
- 第 6 步提供 3 个前端 Mock 模型：`gpt-5.4-mini`、`gpt-5.5`、`gpt-5.6-sol`，默认 `gpt-5.5`。
- 模型选择写入当前浏览器草稿并显示在最终确认页；本轮不修改 API 类型、生成请求或后端模型配置，实际生成仍使用后端默认模型。
- 模型步骤明确标注“预览配置”，避免用户误以为选择已作用于真实生成。

## 示例报告

- “简历改写建议”保持静态 HTML 的左右 BEFORE / AFTER 结构和浅橙、浅绿语义色。
- “高频追问”每题补充样本依据，显示近期示例面经中的出现次数与命中比例。
- 依据文案明确属于“示例面经统计”，不把静态示例数据表述为实时线上结果。

## 报告中心

- 删除 `DATA OWNERSHIP / 私有` 指标卡。
- KPI 区改为 3 列，并保持移动端单列响应式布局。

## 测试与验收

- 前端源码契约测试覆盖 7 步向导、模型 Mock 不进入 API 请求、键盘提示删除、报告中心指标删除和问题依据文案。
- 运行 `npm test` 与 `npm run build`。
- 浏览器验证配置页桌面布局、7 步切换、模型确认展示、示例报告两处内容和报告中心 3 个 KPI。
- 验证 `/ai-analysis` 首页右侧报告预览内容未发生修改。

## 不做

- 不修改后端模型白名单、OpenAI 请求参数或数据库报告模型字段。
- 不新增第三方依赖。
- 不重构与批注无关的 AI 分析业务流程。
