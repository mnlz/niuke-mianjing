# AI 分析静态稿迁移到 React 设计

## 目标

以用户提供的 `home.html`、`create.html`、`sample.html` 和 `reports.html` 为视觉基准，更新现有 4 个 AI 分析 React 页面，同时保留登录、真实数据、简历解析、报告生成和账号报告读取能力。

## 页面映射

| 静态稿 | React 页面 | 路由 |
| --- | --- | --- |
| `home.html` | `AIAnalysis/index.tsx` | `/ai-analysis` |
| `create.html` | `AIAnalysis/CreatePage.tsx` | `/ai-analysis/create` |
| `sample.html` | `AIAnalysis/SampleReportPage.tsx` | `/ai-analysis/sample-report` |
| `reports.html` | `AIAnalysis/ReportsPage.tsx` | `/ai-analysis/reports` |

## 不可修改区域

首页现有右侧报告预览必须保持当前实现和内容不变：

- 容器：`.ai-report-sample`
- 内容容器：`.sample-body`
- 保留“全景求职研判”“字节跳动 · 人工智能/算法”、83 分、3 项补强、招聘信号、面试重点和底部进度条。
- 不用 `home.html` 中新的 `preview-card` 替换该区域。

允许调整首页网格尺寸和外围间距，但不得修改上述组件的 JSX、文案、指标或内部样式。

## 首页

- 采用 `home.html` 的紧凑产品导航、版本标识、用户胶囊、主标题层级和蓝色强调。
- 保留当前首页真实统计数据读取；视觉上增加趋势或时间说明时使用中性文案，不伪造后端字段。
- 雷达图、内测反馈、六种分析模式和底部 CTA 按静态稿重排。
- 报告类型入口继续传递当前 `report` query 参数。
- 未登录用户继续可以浏览首页和示例报告；生成入口沿用现有登录检查链路。

## 配置页

- 采用 `create.html` 的 6 步侧栏、步骤说明、卡片、键盘提示、实时预览和操作栏视觉。
- 继续从 API 读取公司、岗位方向和关联面经，不使用静态稿中的岗位数量。
- 保留当前分析类型规则、公司单选或多选、招聘类型兼容处理、最多 8 篇面经、PDF 上传解析、粘贴文本、联系方式识别、编辑后撤销确认和生成校验。
- 简历型报告必须先确认解析内容；生成成功后继续进入当前账号报告中心。
- 键盘左右键和 Enter 仅在不影响输入框编辑时切换步骤。

## 示例报告

- 用 React 结构呈现 `sample.html` 的完整固定示例，不请求后端。
- 包含封面、综合分、4 项指标、核心判断、能力矩阵、岗位要求与简历证据、简历改写、面试准备和 7 天行动计划。
- 保留复制、打印和“生成我的报告”入口。

## 报告中心

- 采用 `reports.html` 的标题、KPI、分类侧栏、搜索排序、报告表格和详情抽屉视觉。
- 报告列表、数量、类型、分数可用字段和日期全部来自当前账号 API；不注入静态假报告。
- 当前账号没有报告时继续显示真实空状态和“新建第一份报告”。
- 查看、Markdown 下载和删除继续使用现有逻辑；删除仍需确认。
- 静态稿中没有后端来源的数据字段不展示，不用假值补齐。

## 公共导航和样式

- `AnalysisHeader` 统一迁移静态稿的品牌、版本标识、导航、用户胶囊和新建入口。
- 登录状态继续由 `UserSessionButton` 管理；不复制静态用户名。
- 主要页面样式继续集中在 `AIAnalysis/style.css`，不增加 CSS 框架、图表库或新依赖。
- 桌面、平板和手机布局保持可用；移动端不得横向溢出。

## 测试与验收

- 更新 AI 分析纯函数或静态结构契约测试，保护路由、类型和不可修改的首页报告预览文案。
- 运行 `npm test` 和 `npm run build`。
- 浏览器逐页检查首页、六步配置、示例报告、账号空报告状态与已有报告详情。
- 检查控制台错误，并在 390 px 宽度验证无横向溢出。

## 非目标

- 不修改后端接口、数据库结构、AI prompt 或登录规则。
- 不把静态稿假数据写入数据库或报告 API。
- 不重构 AI 分析以外的前端页面。
