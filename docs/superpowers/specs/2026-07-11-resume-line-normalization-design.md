# 简历软换行与编辑区字体设计

## 目标

修复 PDF 版式提取产生的句中硬换行，并提高简历结构化编辑区中偏小文字的可读性。

## 根因

`pypdf` 的 layout 模式会按 PDF 文本块宽度在约 60–70 个字符处切行。后端 `_clean_lines()` 保留这些换行，`_split_sections()` 再将其写入分区内容；前端按换行逐条渲染，因此断句同时影响展示、编辑文本和 AI 分析输入。

## 后端设计

- 保留 layout 模式，因为它对简历分区标题和字段顺序更稳定。
- 在 `_clean_lines()` 之后增加 `_join_wrapped_lines(lines)`。
- 仅当上一原始行长度至少 55、未以句末标点结束，且下一行不像分区标题、编号条目、短标签或常见新条目开头时合并。
- 使用上一原始行长度而非合并后的累计长度，避免连续误吞短条目。
- 用 5 份本地简历验证姓名、联系方式、页数、分区与内容均正常，并为典型断句添加精确断言。

## 前端字体

- 分区标签标题调整到 12px，序号和数量调整到 10px。
- 分区卡片标题调整到 13px，编辑按钮调整到 12px。
- 编辑器辅助说明调整到 11px，字段标签调整到 10px。
- 分区编辑文本调整到 13px。
- 右侧实时预览维持紧凑字号，不扩大。

## 验证

- 后端运行 `pytest tests/test_resume_parser.py`。
- 前端运行 `npm test` 与 `npm run build`。
- 上传操作仍由用户手工完成。
